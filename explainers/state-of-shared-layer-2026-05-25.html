03 What I'm not claiming

Every increment got an internal Devil's Advocate pass. The holes are written down — in the code, the commits, and here — not buried. None block a tested reference; all are tracked for the hardening before this touches live data.

  • The physical wall is proven by content + permissions, not cross-user denial yet — that last step needs each client file owned by its own OS user on the Mac Mini (a one-line deploy step), documented, not faked.
  • Secret scrubbing is best-effort — it catches known shapes (keys, tokens, emails); the real safety net is that claims are quarantined and human-reviewed before anything routes.
  • The runner is at-least-once — handlers must be safe to run twice; one-runner-per-agent is the deploy assumption.
  • Schema checking is opt-in and a JSON-Schema subset — production routes writes through the validated path and may swap in a full validator behind the same surface.
  • Codex review is intentionally deferred to just before deployment — six scoped prompts are drafted, to run against the real ported code, not throwaway prototype specifics.
