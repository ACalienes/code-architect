Then: harden the sync script so this can't rot again

Today it does: git add -A && commit any local change, then git pull (merge). On a diverged repo that's an instant mess — and it wasn't even executable, so it never ran.
Hardened: make it executable, fast-forward-only (never auto-merge), skip + warn on diverged repos instead of committing cruft, branch-aware, and add the missing live repos (framer, kmg).
