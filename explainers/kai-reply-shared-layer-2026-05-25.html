03 The big decision — Kai recommends a cleaner path

✓ Recommendation (Kai + the infra reality)

Shared Layer becomes the one system; old mesh gets a thin adapter + a shut-off date — not "build on top of it."

Why not build on top: the old mesh's routing model is exactly what Codex made us move away from (the per-client isolation redesign). Layering the new system on the old one would drag that flawed model back in.

Why a clean cutover is safe: there's barely any healthy mesh to protect — one solid two-way link (ACD↔Kai) and a couple of one-way flows; everything else is already broken. So we keep a small compatibility shim for just that one live link, set a sunset date, and let the migration *force* the broken routes to finally get fixed.
