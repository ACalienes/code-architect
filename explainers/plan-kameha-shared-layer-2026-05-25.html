02 The rules the design follows

◆Repos stay separate
No merging. Each agent keeps its own codebase and autonomy — that's why they're good at their jobs.
◆One shared channel
Exactly one place everyone reads and writes. Kill the accidental extra paths that cause confusion.
◆Durable by default
Written to disk the instant it's created. Survives a sleeping/waking fleet — nothing rots in a queue.
◆Everyone checks often
Each agent re-reads the shared channel on a fast schedule (~60s). This is where "speed is of the essence" lives.
◆Governed
Keep the good parts of today's mesh — who's allowed to talk to whom, and an audit trail of every message.
◆Typed memory
Not just messages — structured facts ("client approved X") that stay queryable later. The shared brain.

03 The shape — a shared brain + scheduled drainers

One shared store on the Mac Mini that every agent reads and writes. Each agent runs a tiny scheduled job — a drainer — that checks the store for anything addressed to it and acts. Governance (who-can-talk-to-whom + audit) is a shared rulebook that runs inside that read/write, not a separate service that can be down.

05 What we keep, what we retire

Keep (the mesh got these right)

  • Route permissions — who's allowed to message whom, with tiers
  • The audit trail — every message logged
  • The durable filesystem drop as the storage backbone
  • Idempotency — the same fact can't be processed twice

Retire (sources of the mess)

  • The accidental dashboard relay path (a third pipe nobody chose)
  • Drainers that exist but were never put on a schedule
  • Governance as a separate HTTP service that can be down → make it a shared rulebook instead
  • Agents (like CA) silently left out of the routing table

06 How we roll it out (de-risked, in stages)

PHASE 0

Stop the bleeding

Wire every agent (incl. CA) into the routing table, put the existing drainers on a schedule, and retire the accidental relay. Small, immediate, high-relief.

PHASE 1

The shared store + the pilot

Stand up the typed fact tables + the governance rulebook, and prove the whole loop on ONE real flow: the DAG → ACD/NAMI feedback you just lived. Small enough to verify, real enough to trust.

PHASE 2

Everyone on the bus

Migrate all agents to read/write the shared store; add topic subscriptions so an agent gets everything about, say, a campaign — not just direct messages.

PHASE 3

The brain answers questions

Agents query the shared memory on demand ("what did the client say about X?"). The fact store becomes the fleet's long-term knowledge.

08 Decisions you made this round

✓How facts find the right agents
Not discipline, not broadcast. Typed facts + each agent subscribes to what it cares about + a routing map fans out by fact-type. Push what's relevant, pull what's needed.
✓Backfill our history
A one-time ingest seeds the store from weeks of logs, intakes, memory + decisions — so the fleet starts with shared history, not blank.
✓Client repos are in
DAG, TDB et al. contribute and learn — but each sees only its OWN client's data. Cross-client wisdom flows only as anonymized "learnings."

09 What the adversarial review flagged (DA: REVISE)

The CA-internal DA passed the architecture but demanded 4 things be hardened before we build. These are folded into the design; Codex will stress them next.

R1 · No silent drops. A fact that can't be routed must hit a dead-letter queue that alerts you — never vanish. (Our oldest enemy: silent failure.)
R2 · Client isolation is sacred. Default-deny reads (an agent only ever sees its authorized clients) + a human gate on any cross-client share. No auto-sharing on trust.
R3 · Backfill is quarantined. Imported history is scrubbed for secrets/PII and reviewed before any agent can act on it.
R4 · One store, not two. Extend the existing shared database rather than build a parallel one — or we recreate the very mess we're killing.
Also watching: single-point-of-failure backup story, drainer cost (cheap triage before any AI call), out-of-order delivery, SQLite contention. Carried into build.

10 What Codex caught — and the one big correction

Codex's verdict: Phase 0 is sound, but Phase 1 wasn't safe as first drawn. It found a real hole in my isolation plan — and it was right. Here's the honest correction:

The fix I had to make: I'd planned to keep clients apart with a shared "rulebook" that every agent applies when reading. Codex showed that's just a promise — any stray script or stale copy reading the database directly would see everything. Isolation now works by what an agent can physically reach: a trusted router writes each agent its own private inbox, and client repos read only their own pre-filtered slice — they literally cannot see another client's data. A boundary, not a promise.
✓Private inboxes
Each agent reads only rows written for it — not the shared pool. DAG can't reach TDB's data even by accident.
✓No cross-client sharing in v1
"Anonymized learnings" are too easy to re-identify in a 16-client world. Pulled until there's a real redaction regime.
✓Backfill = claims, not gospel
Imported history is tagged as unverified and can be superseded — a stale note can't poison the fleet permanently.
✓Reject bad data at the door
Missing client scope or an unknown fact-type is refused on write, not discovered later.
✓Corrections are first-class
You can say "that fact was wrong" and agents receive the correction — not just an ignored new note.
✓One transport, declared
We name the canonical channel up front so we don't quietly build a second mesh beside the old one.
