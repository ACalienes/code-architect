How it's wired (the safe shortcuts)

  • Routing — CFO subscribes to supervisor_decision; every subscriber gets all of them, so the handler self-filters by subject authorship (acts only on its own draft).
  • Trust without a new endpoint — the supervisor embeds a trusted context{subject_source_agent, draft_ref}; safe because only the alex+supervise token can publish that fact type (the Phase-2 forgery gate).
  • Producer fix — the cfo-draft fact now carries draft_ref so the approval names the exact file. Idempotency key unchanged → no re-posts.
  • Idempotent replay — distinct alex:sd: key prefix avoids colliding with legacy decisions; re-clicking Approve is a clean 200, not a 409.
