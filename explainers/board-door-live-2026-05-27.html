What the door guarantees

✓
No impersonation

Identity comes from the token, server-side — a caller can’t pretend to be another agent.

✓
Closed by default

Tailscale-only, token-required, and it refuses to run if no token is enrolled. No open write hole.

✓
Validated input

Every fact is shape-checked and sanitized before it touches the Board; junk is rejected at the door.

✓
Safe retries

A network retry can’t double-post — the door recognizes and de-duplicates it.

✓
Seeing ≠ doing

Posting a work-order is just a recorded fact — nothing executes. The approval gate is untouched.

✓
Nothing else disturbed

Deployed additively — the running fleet kept humming; all 15 agents stayed online.
