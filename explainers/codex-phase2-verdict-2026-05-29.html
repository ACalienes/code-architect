What Codex found — in plain English

P0 · critical

1. A client-locked agent could touch another client's items

design intended this

An agent token locked to one client is correctly blocked from reading another client's inbox — but the claim / ack / close actions forgot the same lock. If it knew an item's ID, it could close out another client's item. (The design said the client-lock should cover ack too; the code only put it on read.)

Fix: apply the same client check to all four endpoints + add tests that prove cross-client actions are denied.

P0 · critical

2. The “only Alex can approve” guard has a back door

DA flagged the root

The front door (the gateway) correctly refuses to let anyone but Alex publish an approval. But an older, separate write path doesn't enforce that — so an agent could forge an “Alex approved this” through the back. Codex confirmed a fake approval persists and routes. (Our own design review had flagged that this old path trusts whoever it's told — the code didn't close it.)

Fix: make approval-grade facts impossible to create anywhere except the authenticated gateway/supervisor path. Block them in the old path outright.

P1 · high

3. A “dead / needs-review” item can be secretly revived

When an item fails and gets quarantined (parked for human review), a late ack can flip it back to “done” — hiding a failure that should have stayed visible to you.

Fix: ack must refuse dead items (return “gone”), and we test the quarantine-then-ack ordering.

P1 · high

4. The “only one runner” lock isn't airtight for duplicate copies

If two copies of the same agent's consumer run at once (same token), both could grab the same item and both act on it — a double-send risk (e.g. an invoice sent twice).

Fix: give each running copy its own claim ID and only let that exact owner renew — a true compare-and-swap lock.

P2 · lower

5. Corrupted permission data defaults to “allowed”

If a token's permissions field is broken (garbled, not just empty), the code treats it as “publish allowed” instead of denying. Fails open instead of closed.

Fix: empty/legacy = publish-only (fine); broken = deny everything.
